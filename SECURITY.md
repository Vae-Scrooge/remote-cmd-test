## 报告安全漏洞

Remote CMD 项目非常重视安全问题。如果您发现了安全漏洞，请不要在公共 Issue 中报告。

### 报告流程

1. **发送邮件到安全团队**
   - 邮箱：security@example.com
   - 主题：[SECURITY] 漏洞描述

2. **邮件内容应包括：**
   - 漏洞类型（如：认证绕过、命令注入等）
   - 受影响版本
   - 详细复现步骤
   - 可能的修复建议
   - 您的联系方式（可选）

3. **响应时间**
   - 确认收到：24 小时内
   - 初步评估：72 小时内
   - 修复计划：1 周内

### 负责任的披露

- 请给我们合理的时间来修复问题
- 在公开披露前，我们会与您协调发布时间
- 修复完成后，我们会在 CHANGELOG 中致谢报告者

## 安全最佳实践

### 1. SSH 密钥管理

- 使用 SSH Key 而非密码
- 密钥文件权限设置为 600
- 定期轮换密钥
- 使用密钥管理工具（如 ssh-agent）

### 2. 配置文件保护

- 不要将配置文件提交到版本控制
- 使用 .gitignore 排除敏感文件
- 配置文件权限设置为 600
- 使用环境变量存储密码

### 3. 最小权限原则

- 使用具有最小必要权限的用户
- 避免使用 root 账户
- 使用 sudo 执行特权命令
- 记录所有特权操作

### 4. 网络安全

- 使用防火墙限制 SSH 访问
- 考虑使用 VPN 或跳板机
- 启用 SSH 日志审计
- 监控异常连接
